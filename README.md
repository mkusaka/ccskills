# ccskills

Piebald's Claude Code system prompt repository can be turned into installable `skills` packages, but `npx skills add` only installs directories that contain `SKILL.md` files with YAML frontmatter. This repo adds the missing normalization layer and keeps the generated skills under [`skills/`](/Users/masatomokusaka/src/github.com/mkusaka/ccskills/skills).

## What Was Confirmed About `npx skills add`

- The CLI accepts GitHub shorthand, full GitHub URLs, GitHub tree URLs, generic git URLs, and local paths.
- The implementation also accepts `owner/repo@skill`, `owner/repo/subpath`, `github:owner/repo`, and non-git HTTP(S) URLs via `/.well-known/skills/index.json`. Source code is the reliable reference here; the README is slightly behind the implementation in a few places.
- `owner/repo@skill-name` is supported in the implementation, even though the README focuses mostly on `--skill`.
- Discovery is `SKILL.md`-based. The Vercel CLI looks for directories containing `SKILL.md` and requires YAML frontmatter `name` and `description`.
- For this upstream repo, the only files that are actual built-in skills are the `skill-*.md` files listed in the upstream README's `## Skills` section. The wider `*skill*.md` set also includes tool descriptions and system reminders, which are skill-adjacent but not normal end-user skills.

## Generated Output

The generated directories in [`skills/`](/Users/masatomokusaka/src/github.com/mkusaka/ccskills/skills) are what `npx skills add mkusaka/ccskills` will discover.

By default this repo syncs only upstream `skill-*.md` files:

- `build-with-claude-api`
- `build-with-claude-api-reference-guide`
- `create-verifier-skills`
- `debugging`
- `loop-slash-command`
- `simplify`
- `update-claude-code-config`
- `verification-specialist`

If you want the broader "skill-related" prompt set as installable skills too, the sync script also supports that explicitly.

## Usage

```bash
pnpm sync
```

The default source is:

```text
https://github.com/Piebald-AI/claude-code-system-prompts/tree/main/system-prompts
```

Optional modes:

```bash
# Include broader skill-adjacent prompts such as tool descriptions and reminders
pnpm sync:related

# Dry-run without writing files
pnpm sync --dry-run

# Point at a different source
pnpm sync Piebald-AI/claude-code-system-prompts
pnpm sync /path/to/local/claude-code-system-prompts
```

Generated skills can then be installed with the Vercel CLI:

```bash
npx skills add mkusaka/ccskills
npx skills add mkusaka/ccskills@debugging
npx skills add https://github.com/mkusaka/ccskills/tree/main/skills/debugging
```

## Automation

This repo includes [`.github/workflows/sync-piebald-skills.yml`](/Users/masatomokusaka/src/github.com/mkusaka/ccskills/.github/workflows/sync-piebald-skills.yml), which runs daily at `03:17 UTC` and on `workflow_dispatch`.

The workflow:

- runs `pnpm sync`
- runs `pnpm test`
- if generated files changed, runs [`scripts/sync_skills_pull_request.sh`](/Users/masatomokusaka/src/github.com/mkusaka/ccskills/scripts/sync_skills_pull_request.sh)
- pushes `automation/piebald-skill-sync`
- creates or updates a PR with `gh pr create` / `gh pr edit --body-file`
- verifies the rendered body with `gh pr view --json body --jq .body`

For GitHub Actions, make sure the repository allows `GITHUB_TOKEN` write access for contents and pull requests.

You can also dry-run the PR script locally:

```bash
SYNC_PR_DRY_RUN=1 pnpm sync:pr
```

## Implementation Notes

- This importer is intentionally narrower than `npx skills add`: it currently supports GitHub shorthand/URLs and local paths, which is enough for the Piebald sync flow.
- The sync script uses the GitHub Contents API for remote sources and can also read a local checkout.
- Source metadata from Piebald's HTML comment header is preserved under `metadata` in each generated `SKILL.md`.
- Sync state is tracked in `skills/.ccskills-sync.json`, so reruns only delete directories that were previously generated by this repo's sync process.
